# Hoja de ruta de Aulafy

Última actualización: 14 de julio de 2026.

Esta hoja de ruta expresa prioridades, no promesas ni fechas cerradas. Una función solo se considera disponible cuando está implementada, verificada y documentada en la rama principal.

## Visión

Convertir Aulafy en una referencia abierta en español para aprender inteligencia artificial de forma práctica, verificable y responsable, desde la alfabetización inicial hasta la construcción y operación de sistemas reales.

## Criterios de prioridad

Una propuesta tiene más prioridad cuando:

1. mejora un resultado educativo observable;
2. corrige información insegura, imprecisa o desactualizada;
3. hace una práctica más reproducible o fácil de verificar;
4. reduce riesgos de privacidad, permisos, costes o dependencia de proveedor;
5. mejora accesibilidad, navegación o comprensión;
6. beneficia a varias rutas o cursos sin duplicar contenido;
7. incluye fuentes, pruebas y una forma clara de mantenerla.

## Ahora: consolidar la base

- [ ] Revisar cada curso con una ficha homogénea de público, requisitos, duración, objetivos y entregable.
- [ ] Auditar ejemplos críticos para confirmar comandos, versiones, licencias y resultados esperados.
- [ ] Eliminar duplicaciones entre páginas antiguas, contenido estructurado, PDFs e índices derivados.
- [ ] Unificar el vocabulario: código abierto, pesos abiertos, servicio comercial, modelo local y agente.
- [ ] Mantener la paridad de significado entre español e inglés en cambios críticos.
- [ ] Incorporar plantillas de incidencias y propuestas en GitHub.
- [ ] Automatizar en integración continua las verificaciones principales y el análisis de secretos.

## Siguiente: mejorar la experiencia de aprendizaje

- [ ] Añadir diagnósticos iniciales para recomendar una ruta según experiencia, hardware y objetivo.
- [ ] Reforzar los proyectos finales con rúbricas, evidencias y listas de revisión descargables.
- [ ] Mostrar con más claridad prerrequisitos y dependencias entre cursos.
- [ ] Mejorar búsqueda, filtros y continuidad entre lecciones relacionadas.
- [ ] Ampliar ejemplos para Windows, macOS y Linux sin ocultar diferencias importantes.
- [ ] Realizar una revisión sistemática de accesibilidad de navegación, contraste, teclado y lectores de pantalla.
- [ ] Añadir un registro editorial de cambios relevantes por curso.

## Después: comunidad y materiales reproducibles

- [ ] Publicar proyectos de referencia pequeños que puedan ejecutarse, probarse y bifurcarse.
- [ ] Crear un proceso de revisión comunitaria para fuentes, ejemplos y proyectos finales.
- [ ] Ofrecer paquetes descargables por ruta con ejercicios, plantillas y criterios de evaluación.
- [ ] Explorar un modo de lectura sin conexión que no requiera una cuenta central.
- [ ] Incorporar evaluaciones locales de bajo riesgo para comprobar conocimientos y resultados.
- [ ] Documentar versiones estables de cursos cuando una herramienta cambie de forma incompatible.

## Líneas permanentes

Estas tareas no tienen una fecha de finalización:

- revisar enlaces y documentación oficial;
- retirar ejemplos inseguros u obsoletos;
- comprobar dependencias y secretos;
- corregir errores de idioma y accesibilidad;
- mantener el contenido gratuito y reutilizable;
- explicar límites, costes, privacidad y alternativas locales;
- atender propuestas y reconocer contribuciones.

## Fuera de alcance por ahora

- cuentas de alumno y perfiles centralizados;
- venta de cursos o suscripciones;
- certificados que pretendan equivaler a una acreditación oficial;
- automatizaciones que actúen sobre sistemas reales sin revisión ni permisos explícitos;
- contenido generado en volumen sin revisión editorial;
- dependencia obligatoria de un único proveedor de IA.

## Cómo proponer una prioridad

Abre una incidencia explicando el problema educativo, a quién afecta, qué evidencia tienes y cómo se podría verificar la mejora. Para propuestas privadas o que incluyan seguridad, escribe a [learntouseai@gmail.com](mailto:learntouseai@gmail.com).
